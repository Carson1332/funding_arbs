## Project Overview

`funding-arb` is a **delta-neutral crypto funding rate arbitrage** research framework with trackable backtest results. It collects 2+ years of 8-hour funding rate data across 50 perpetual pairs on **Binance**, **Bybit**, and **OKX**, then backtests a strategy that goes long spot and short perpetual to harvest funding payments.

The strategy uses composite signals (funding z-score, basis momentum, open-interest concentration, term structure) and a **Kalman filter** for dynamic hedge ratio estimation. The backtest engine replays historical 8-hour funding snapshots with realistic fees, slippage, and risk management.

Key documentation lives in:
- `README.md` — quick start, project structure, badges
- `docs/architecture.md` — system layers and data flow
- `docs/strategy.md` — plain-English strategy description

## Code Style Guidelines

### Language and Spelling
- Write docstrings and comments in **English**.
- The existing docs use a mix of British and American spelling; match the file you are editing.

### Imports
- Every module must start with `from __future__ import annotations`.
- Group imports: stdlib → third-party → local.

### Type Hints
- Use type hints for function signatures and important variables.
- Use `Optional[...]` and `from __future__ import annotations` to avoid string-forward-references.

### Docstrings
- Use Google-style docstrings with explicit `Parameters` and `Returns` sections.
- Include units where applicable (e.g., "funding rate as decimal", "basis points").

### Config / Parameters
- Encapsulate tunable parameters in `@dataclass` objects (e.g., `ZScoreParams`, `SimulatorConfig`).
- Default values belong in the dataclass, not scattered in function bodies.

### DataFrame Contracts
- Signal modules take a `pd.DataFrame` with known columns and return an augmented `pd.DataFrame`.
- Common required columns: `timestamp`, `symbol`, `exchange`.
- Always sort by `[symbol, exchange, timestamp]` before processing.

### CLI Entry Points
- Modules intended to be run directly expose a `main()` function guarded by `if __name__ == "__main__":`.
- Accept `--config` argument defaulting to `config/default.yaml`.

### Naming Conventions
- Classes: `PascalCase`
- Functions / variables / modules: `snake_case`
- Constants: `UPPER_SNAKE_CASE`
- Private helpers: leading underscore `_helper()`

- **pytest** with class-based test suites.
- Synthetic data generators (e.g., `_make_funding_df`, `_make_test_data`) are preferred over network calls.

### Coverage Areas

- `test_kalman.py` — simple hedge ratio (always runs); Kalman tests skip if `filterpy` missing
- `test_fee_model.py` — fee calculations, funding payments, custom schedules
- `test_fee_model.py` — fee calculations, funding payments, custom schedules

### Running Tests
```bash
pytest tests/ -v --tb=short        # fast feedback
pytest tests/ --cov                # with coverage
```

### Skipping Optional Dependencies
- Tests that require `filterpy` must use `pytest.skip("filterpy not installed")` when the import fails.
- The backtest engine itself falls back to `SimpleHedgeRatio` if `filterpy` is unavailable.

## Key Architectural Patterns

### Layered Data Flow
```
Exchange APIs → Data Layer → Research Layer → Backtest Engine → Report Layer
(ccxt/CoinAPI)   (Parquet)    (signals)        (simulation)      (metrics/tearsheet)
```

### Signal Composition
Each research module is a **standalone signal generator**:
- Input: `pd.DataFrame` of market data
- Output: augmented `pd.DataFrame` with signal columns (typically `signal: int ∈ {-1, 0, 1}`)
- Signals can be chained or swapped without touching the backtest engine.

### Backtest Engine (`enhanced_engine.py`)
The enhanced engine replays historical 8-hour funding snapshots chronologically
for each strategy variant (adaptive carry, term structure, cross-exchange, etc.).
It uses correct carry-trade accounting where capital is locked as collateral
rather than spent, and supports signal-strength-based position sizing.

### Caching Strategy
- All downloaded data is cached as **Parquet** under `data/cache/` subdirectories.
- Subsequent runs perform incremental updates (fetch only data newer than the latest cached timestamp).
- `data/cache/` is **`.gitignore`d**; never commit raw market data.

## Configuration

`config/default.yaml` centralises all parameters. Key fields:

```yaml
zscore_entry: 1.5               # z-score threshold to enter
zscore_exit: 0.0                # z-score threshold to exit
min_annualised_rate: 0.05       # minimum 5% annualised funding
initial_capital: 100000         # starting capital (USDT)
max_positions: 10               # max concurrent positions
max_position_pct: 0.15          # max 15% of equity per position
max_drawdown_exit: 0.10         # exit all if drawdown > 10%
use_kalman_hedge: true          # enable Kalman filter
rebalance_threshold: 0.03       # 3% drift triggers hedge rebalance
```

`config/universe.yaml` defines the 50 perpetual pairs (ccxt unified symbols like `BTC/USDT:USDT`).

## CI/CD Pipeline

### `ci.yml` (Pull Requests & Main)
- Runs on: `pull_request` to `main`, `push` to `main`
- Steps: checkout → install deps → `ruff check .` → `mypy` (continue-on-error) → `pytest`

### `backtest.yml` (Weekly + Path Triggers)
- Runs on: Sunday 06:00 UTC cron, push to `research/**`, `backtest/**`, `config/**`, or manual dispatch
- Steps:
  1. Install deps
  2. Download latest data (`python -m data.downloader`)
  3. Run parameter sweep (`python run_parameter_sweep.py`)
  4. Extract metrics from `results/metrics.json`
  5. Update BYOB badges (Sharpe, Annual Return, Max DD)
  6. Commit `results/` to repo (`[skip ci]`)
  7. Publish tearsheet to GitHub Pages (`results/` → `reports/`)

## Security Considerations

- **Never commit API keys.** Use environment variables:
  - `COINAPI_KEY` — optional premium data source
  - `DATABASE_URL` — optional database connection string
  - `DB_PASSWORD` — used by Docker Compose PostgreSQL
- `.env` and `.env.*` are `.gitignore`d.
- Raw market data in `data/cache/` is `.gitignore`d.
- The CI workflow has `contents: write` and `pages: write` permissions only; it uses `secrets.GITHUB_TOKEN`.

## Common Pitfalls for Agents

1. **Do not commit Parquet or SQLite files.** They are explicitly `.gitignore`d for data privacy.
2. **Do not hard-code fee schedules.** Add new exchanges via `EXCHANGE_FEES` in `backtest/fee_model.py`.
3. **Preserve DataFrame contracts.** Research modules must return DataFrames with the same row count plus new columns; dropping rows silently breaks downstream signal composition.
4. **8-hour intervals are the atomic time unit.** Converting to daily or hourly requires careful re-aggregation; annualised rates assume 3 periods/day (`rate * 3 * 365`).
5. **Kalman filter is optional.** Always provide a fallback (e.g., `SimpleHedgeRatio`) when `filterpy` is absent.
6. **Runner orchestration.** `run_parameter_sweep.py` and `run_single_config.py`
   orchestrate data loading → feature computation → simulation → reporting.
